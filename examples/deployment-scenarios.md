quickly with default settings:\n\n```bash\n# Install the CLI\npip install zscaler-mcp-deployer\n\n# Configure AWS credentials\naws configure\n\n# Run interactive deployment\nzscaler-mcp-deploy deploy\n```\n\nThe CLI will prompt for:\n- Runtime name (e.g., `zscaler-mcp-prod`)\n- Secret name (defaults to `zscaler/mcp/credentials`)\n- Role name (defaults to `zscaler-mcp-execution-role`)\n- Zscaler credentials\n\n## Production Deployment\n\nFor production environments with explicit configuration:\n\n```bash\n# Deploy with all parameters specified\nzscaler-mcp-deploy deploy \n  --runtime-name zscaler-mcp-production \n  --secret-name zscaler/mcp/prod/credentials \n  --role-name zscaler-mcp-prod-role \n  --region us-east-1 \n  --description "Production Zscaler MCP server" \n  --enable-write-tools \n  --non-interactive\n\n# Verify deployment\naws bedrock get-agent-core-runtime \n  --agent-core-runtime-id <runtime-id-from-output> \n  --region us-east-1\n```\n\n## Deployment\n\nFor CI/CD pipelines and automated environments:\n\n```bash\n# Pre-validate environment\nzscaler-mcp-deploy preflight \n  --region us-east-1 \n  --zscaler-cloud zscaler \n  --skip-zscaler  # Skip interactive credential validation\n\n# Deploy with all parameters from environment variables\nzscaler-mcp-deploy deploy \n  --runtime-name $DEPLOYMENT_NAME \n  --secret-name zscaler/mcp/$ENVIRONMENT/credentials \n  --role-name zscaler-mcp-$ENVIRONMENT-role \n  --region $AWS_REGION \n  --description "$ENVIRONMENT Zscaler MCP server', '--', 'enable-write-tools \n  --non-interactive \n  --poll-timeout 900 \n  --verification-timeout 180\n\n# Check deployment status\nif [ $? -eq 0 ]; then\n  echo "Deployment successful"\nelse\n  echo "Deployment failed', 'exit 1\nfi\n```\n\n## Troubleshooting Deployment\n\nWhen deployments fail, use these approaches:\n\n### Quick Redeployment\n\n```bash\n# Run preflight first to identify issues\nzscaler-mcp-deploy preflight --region us-east-1\n\n# Deploy with increased timeouts for troubleshooting\nzscaler-mcp-deploy deploy \n  --runtime-name zscaler-mcp-troubleshoot \n  --secret-name zscaler/mcp/troubleshoot/credentials \n  --role-name zscaler-mcp-troubleshoot-role \n  --region us-east-1 \n  --poll-timeout 1200 \n  --verification-timeout 300 \n  --non-interactive\n```\n\n### Minimal Deployment for Testing\n\n```bash\n# Skip verification for faster iterations\nzscaler-mcp-deploy deploy \n  --runtime-name zscaler-mcp-test \n  --secret-name zscaler/mcp/test/credentials \n  --role-name zscaler-mcp-test-role \n  --region us-east-1 \n  --skip-verification \n  --non-interactive\n```\n\n### Manual Cleanup\n\nIf automated cleanup fails:\n\n```bash\n# Delete runtime manually\naws bedrock delete-agent-core-runtime \n  --agent-core-runtime-id your-runtime-id \n  --region us-east-1\n\n# Delete role manually\naws iam delete-role \n  --role-name zscaler-mcp-execution-role\n\n# Delete secret manually\naws secretsmanager delete-secret \n  --secret-id zscaler/mcp/credentials\n```\n\n### Log Analysis\n\n```bash\n# Tail CloudWatch logs for deployment\naws logs tail /aws/bedrock/your-runtime-id \n  --follow \n  --region us-east-1\n\n# Filter for specific error patterns\naws logs filter-log-events \n  --log-group-name /aws/bedrock/your-runtime-id \n  --filter-pattern "ERROR', 'region us-east-1\n```\n\n## Best Practices\n\n### Security\n\n1. Use separate secrets for each environment\n2. Enable write tools only when necessary\n3. Use custom KMS keys for enhanced security\n4. Regular rotation of Zscaler API keys\n\n### Operations\n\n1. Use descriptive names for resources\n2. Tag resources for cost allocation\n3. Monitor CloudWatch logs for errors\n4. Implement proper error handling in automation\n\n### Development\n\n1. Test with `--skip-verification` for faster iterations\n2. Use different regions for isolation\n3. Keep CLI updated to latest version\n4. Document your deployment process\n\n### Troubleshooting\n\n1. Always run preflight before deployment\n2. Check IAM permissions when getting AccessDenied\n3. Monitor CloudWatch logs for runtime issues\n4. Clean up failed resources promptly']
